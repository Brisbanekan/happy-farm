# 開心農場 — 跨裝置雲端同步設定教學

照著做一次，之後手機和電腦就能玩同一個農場、自動同步。
總共三步：**A. 建立免費雲端資料庫 → B. 把網址貼進遊戲 → C. 上傳到 GitHub Pages**。
（A、B 不用付費、不用信用卡。）

---

## A. 建立 Firebase 免費資料庫（拿到一個網址）

1. 用 Google 帳號登入 👉 https://console.firebase.google.com
2. 點 **「建立專案 / Create a project」**，輸入名稱（例如 `happy-farm`）→ 一直按下一步 → Analytics 可以關掉 → **建立**。
3. 左側選單點 **Build → Realtime Database**（注意是「Realtime Database」，不是 Firestore）。
4. 按 **建立資料庫 / Create Database**：
   - 地區選離你近的（例如 `asia-southeast1`）。
   - 安全性規則選 **「以測試模式開始 / Start in test mode」** → 啟用。
5. 進去後上方會看到資料庫網址，長得像：
   ```
   https://happy-farm-xxxx-default-rtdb.firebaseio.com
   ```
   **把這個網址複製起來**（B 步要用）。
6. （重要，讓它長期可用）點上方 **規則 / Rules** 分頁，把內容整段換成下面這個，再按 **發布 / Publish**：
   ```json
   {
     "rules": {
       "farms": {
         ".read": true,
         ".write": true
       }
     }
   }
   ```
   > 說明：只開放 `farms` 這個區塊公開讀寫，你的「農場代碼」是隨機產生、別人難以猜到，對休閒小遊戲來說足夠。測試模式預設 30 天後會自動鎖住，換成上面規則就不會過期。

---

## B. 把網址貼進遊戲

1. 用記事本／文字編輯器打開 `happy-farm.html`。
2. 找到最上面這一行（檔案開頭附近）：
   ```js
   const CLOUD_DB = "";
   ```
3. 把 A 步複製的網址貼進雙引號裡，例如：
   ```js
   const CLOUD_DB = "https://happy-farm-xxxx-default-rtdb.firebaseio.com";
   ```
4. 存檔。

---

## C. 上傳到 GitHub Pages（讓手機也能開）

1. 註冊／登入 👉 https://github.com
2. 右上角 **＋ → New repository**：
   - Repository name 填 `happy-farm`
   - 選 **Public**
   - 按 **Create repository**
3. 在新頁面點 **「uploading an existing file」**（或 Add file → Upload files）。
4. 把改好的 `happy-farm.html` 拖進去。
   - 建議先把檔名改成 **`index.html`**，這樣網址比較短。
   - 按 **Commit changes**。
5. 上方點 **Settings → 左側 Pages**：
   - Source 選 **Deploy from a branch**
   - Branch 選 **main**、資料夾選 **/ (root)** → **Save**。
6. 等 1～2 分鐘，Pages 頁面會出現你的網址，例如：
   ```
   https://你的帳號.github.io/happy-farm/
   ```
   （如果檔名沒改成 index.html，網址後面要再加 `happy-farm.html`）

---

## D. 開始跨裝置同步 🎉

1. **電腦**：打開你的 github.io 網址 → 點底部 **☁️ 同步** → **建立雲端存檔（產生代碼）**。
   - 會得到一組 6 碼「農場代碼」和一個 QR code。
2. **手機**：用瀏覽器打開**同一個** github.io 網址 → 點 **☁️ 同步**：
   - 輸入電腦顯示的代碼按「讀取」，**或**直接用相機掃 QR。
3. 完成！之後兩邊都會**自動雲端存檔**，換裝置打開就接續同一個農場。

---

## 常見問題

- **要兩邊同時玩嗎？** 不建議同時操作，雲端採「最後存檔的覆蓋」；分開玩沒問題。
- **代碼會被別人看到嗎？** 代碼是隨機的，別人沒有代碼就讀不到你的農場。想更換可在另一台重新「建立雲端存檔」。
- **沒設定 CLOUD_DB 會怎樣？** 遊戲照常運作，只是存在本機、不跨裝置（和現在一樣）。
- **改了網址要重貼一次嗎？** 只要 `CLOUD_DB` 填過一次並上傳，之後就一直有效。
- **不想用雲端？** 玩法面板裡的「⬇️ 匯出 / ⬆️ 匯入存檔」仍可手動搬移進度。
